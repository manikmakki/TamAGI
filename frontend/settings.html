⛟ Secret Store API keys & credentials

Secrets are stored in the OS keyring when available (GNOME Keyring, macOS Keychain, Windows Credential Vault), or in a Fernet-encrypted file at data/secrets.enc otherwise. Values are never written to config.yaml, log files, or LLM messages — they are injected as environment variables only at MCP server spawn time.

Reference a stored secret in config.yaml by name: BRAVE_API_KEY: {secret: BRAVE_API_KEY}

⬢ MCP Servers Connected tool servers

MCP servers are configured in config.yaml under the mcp.servers key. Connections are opened at startup — restart TamAGI after adding or removing a server. Each server's tools are automatically registered as skills the model can call.

For servers that require an API key, store the key here (above) and reference it by name in the server's env block. See config.example.yaml for annotated examples.
